##Anforderungen 
Die Gewissen GmbH ist ein Motorenhersteller für PKWs und Nutzfahrzeuge, es existieren viele Umweltauflagen im Breich des Lärmschutzes.
Hierzu liegen zuberücksichtigende Verordnungen und Gesetzte vor. 

Die Lärmquellen bestehen unteranderm aus Maschienen, diese werden dokumentiert. 
Lärmquellen müssen aufgezeichnet, nachverfolgt und regelmäßig gemessen werden. 

Das System soll ein Ersatz für die vorhandene Excel-Tabelle darstellen. 
Hier sind derzeit Messpunktbeschreibungen und Verläufe der Lärmbelastungen dokumentiert.

Das System soll die Standardmessverfahren nach ISO???? (wird von Herrn Schiemann gelifern) berücksichtigen.


##Vorstellung für neues System
Die Nutzer des Systms solle Mitareiter der Firma Gewissen GmbH sein. 
Es sollen die Zeitpunkte und die Dezibelbelastung an vorgegebenen Messstellen dokumentiert werden.
Das zu entwickelne Systen-Tool soll Mehrbenutzer fähig sein. Wobei Nutzer nicht zwangsläufig gleichzeitig auf das System-Tool zugreifen müssen. 
Die Möglichkeit sollte gegebensein, dass verschiedene Leser/Benutzer auf das System zugreifen dürfen. Dafür wird kein extra Rechte und Rollenkonzept benötigt. 
Aus den erfasssten Daten soll ein Geschäftsbericht sowie ein Copliance erstellt werden können. (Fragen)
Regelmäßige Messungen soll berücksichtigt werden.
Die Software soll Momentan nicht erweiterbar sein. Messpunkte müssen hinzugefügt werden und auch Veränderbar sein. 


## Non functional requirements
Es soll ein kleines Tool sein. Das auf einem Windows System laufen soll.
Das System soll an das schon bestehende OpenResKit Ökosystem agebunden sein und auf der bestehen Client-Server Architekur aufbauen. 
Das Tool soll nicht so wie Excel funktionieren. Es soll mehr Überblick verschaffen und an die jeweiligen Unternehmens Anforderungen angepasst werden können.
Das System soll genau am 10. Februar 2015 abgeliefert werden.
Alte Messung (sowohl Digital als auch Analog) müssen Importiert werden können.
